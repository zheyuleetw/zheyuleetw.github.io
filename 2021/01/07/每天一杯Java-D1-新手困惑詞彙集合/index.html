每天一杯Java-D1 新手困惑詞彙集合

發表於 2021-01-07 更新於 2021-04-10 分類於 Backend

開場小聊-那些翻成中文也看不懂的英文

2020年9月底剛從半導體轉軟體時，時常令我困惑的是那些看起來高冷的英文縮寫，散發神秘又高貴的氣息，其實踏入新的領域都無法避免這段用語熟悉陣痛期，然而，我感受到軟體工程的用語特別百花齊放，也特別的抽象，剛接觸程式語言時有海量的英文縮寫要理解，有些是說十遍也說不清的，我的經驗是聽不懂不需心急，先將表面的定義刻在心底，待時光一點一滴琢磨，隨著編程經歷增加會不斷有新的體會的，這裡就回憶一下當初令我頭痛的幾個縮寫，也順便複習與理清我的理解是否正確。

後記，時正搬遷部落格的2021年4月，想起之前在youtube上看的軟體工程:電腦科學的速成課影集，講師Carrie Anne在從電腦硬體(如電晶體的物理原理到記憶體和中央處理器等)循序漸進到軟體的解說過程中就不斷強調*A new level of abstraction!*也就是硬體一直層層包裹成新的元件，例如數個電晶體組合被包裝成另一個成為布爾邏輯的元件，到軟體就是所謂物件導向編程的封裝特性，我覺得電腦科學這麼多的英文縮寫名詞，似乎也體現了這一點，將一長串的文字包裝一個縮寫，賦予新的概念和生命。

A new level of abstraction!

1.API - Application Programming Interface 應用程式介面

從英文理解，application是應用程式，programming是程式設計，interface是介面，聚合起來就是與設計或撰寫應用程式時相關的介面。

從中文理解，對岸翻譯interface為接口，我舉得非常簡潔有力，顧名思義為程式和程式之間對接的窗口，這個窗口定義了可以呼叫的方法、可以請求的種類、如何呼叫方法、如何發出請求、數據的格式以及其他特定的規範。

舉個生活化的例子，想像有一個四周環海的島嶼要與對外聯絡，必須造橋連接大陸，橋就是所謂的API，定義道路的寬度以及來往的規則等，有意想和島嶼建橋連接的城市都需要按照規定建造橋樑，一旦建造好了，符合資格的人就可以來往島嶼，也就是所謂特定的數據格式，重點是人不需要知道橋確切是如何建造的，這就是API的功能，封裝了各種類、方法和定義，方便程式與程式之間的溝通。

具體Java的例子是JDBC API，是Java對資料庫連接所制訂的API，各家資料庫（MySQL、Oracle、DB2等）根據JDBC API寫好驅動程式，工程師就可以透過驅動程式與對應的資料庫連線。

2.OOP - Object Oriented Programming 物件導向編程

對新手來說，第一個知識盲點大概是物件到底是什麼魔神仔，若時光倒流，我回到2020年的9月底，我會告訴自己先記住一句話，萬事萬物皆物件，舉凡一台腳踏車，一個銀行帳戶或是一台ATM，都是物件，在撰寫代碼時，傾向用代碼描述一個物件，包含這個物件的特性與行為，而應用程式的構成就是大大小小的物件以及物件之間互相發送訊息，這就是物件導向編程的思想，以一個銀行帳戶和ATM為例，撰寫代碼時，會描述這個帳戶持有人、存款金額、年息、貸款金額，而ATM則是具有存錢的功能以及領錢的功能

3.JDK/JRE/JVM/ - Java Developmet Kit Java開發工具包

4.JRE - Jave Runtime Environment Java運行環境

5.JVM - Java Virtual Machine Java虛擬機器

JDK是Jue De Kuai （覺得快）的縮寫，意思是安裝了就會覺得開發很快

JDK顧名思義是使用Java語言開發所需要用到的工具包，裡面包含各種Java套件與工具，高頻使用的有java.util, java.lang等，機器裝上JDK就能使用這些套件，而JRE就是當機器需要執行Java程式時所需要的環境，就像魚必須在具有水的環境下才能生存，Java必須在安裝有JRE的機器上才能生存，JVM則是Java底層的執行機器，所以實際上我們是在自己的機器上安裝了一台Java的虛擬機器，這三者之間的關係可以由下面公式理解：

JDK = JRE + Tools & Tool APIs
JRE = JVM + Java SE API
